| 명령어 | 한 줄 요약 | 중요도 |
| :--- | :--- | :---: |
| **top** | 실시간 시스템 모니터링 | ⭐⭐⭐ |
| **ps** | 현재 실행 중인 프로세스 확인 | ⭐⭐⭐ |
| **jobs** | 백그라운드 작업 목록 확인 | ⭐⭐ |
| **kill** | 프로세스 강제 종료 | ⭐⭐⭐ |

*   ### 📊 top
    *   **설명**: 컴퓨터의 CPU와 메모리 사용량을 실시간으로 보여줍니다.
    *   **팁**: `q`를 누르면 화면에서 빠져나올 수 있어요!

*   ### 📝 ps
    *   **설명**: 현재 어떤 프로그램들이 켜져 있는지 스냅샷을 찍듯 보여줍니다.
    *   **자주 쓰는 옵션**: `ps -ef` 또는 `ps aux`

*   ### ⏳ jobs
    *   **설명**: 현재 터미널 세션에서 뒤로 돌린(백그라운드) 작업들의 상태를 보여줍니다.

*   ### 🛑 kill
    *   **설명**: 멈춘 프로그램을 종료할 때 사용합니다.
    *   **예시**: `kill -9 [프로세스 ID]` (강제 종료)

# 과제 2: Python을 활용한 나만의 SW 개발

> **프로그램명**: 용돈 잔소리 비서 (Pocket Money Secretary)
> **개발 언어**: Python 3.x
> **파일명**: `pocket_money.py`

## 1. 개발 동기 및 목적
매달 용돈을 받을 때마다 계획성 없이 지출하여 월말이 되기 전에 재정난을 겪는 문제를 해결하고자 본 프로그램을 기획했다. 단순히 지출 내역만 기록하는 기존 가계부 앱들과 달리, **다음 용돈 지급일까지 남은 일수를 계산하여 하루 권장 지출액을 제시**하고, 이를 초과할 경우 실시간으로 경고 메시지를 보내 지출을 억제할 수 있도록 돕는 실용적인 프로그램을 개발하는 것이 목적이다.

## 2. 주요 기능 및 알고리즘
* **사용자 초기 설정**: 이번 달 총 가용 용돈 금액과 다음 용돈 지급 날짜(`YYYY-MM-DD` 형식)를 입력받는다.
* **디데이(D-Day) 및 하루 예산 자동 계산**: 내장 라이브러리인 `datetime`을 활용해 오늘 날짜와 목표 날짜 사이의 남은 일수를 계산하고, 남은 용돈을 일수만큼 나누어 **'오늘 하루 동안 쓸 수 있는 권장 예산'**을 도출한다.
* **지출 평가 및 잔소리 시스템 (핵심 조건문)**: 사용자가 금일 지출한 금액을 입력하면, 앞서 계산된 하루 권장 예산과 비교하여 총 3단계의 조건문(`if-elif-else`)에 따라 피드백을 출력한다.
  1. **과소비(권장 예산의 1.4배 초과)**: 강력한 비상 경고 및 절약 촉구 메시지 출력
  2. **주의(권장 예산 초과)**: 지출 주의 및 내일 자제 권고 메시지 출력
  3. **절약(권장 예산 이하)**: 절약 성공 칭찬 메시지 출력
* **잔액 업데이트**: 지출액을 차감한 최종 남은 용돈을 사용자에게 직관적으로 보여준다.

## 3. 코드 구현 중 겪은 에러 및 본인 평가
* **날짜 연산 예외 처리 문제**: 처음에 사용자로부터 문자열 형태로 날짜를 입력받았을 때, 오늘 날짜(`datetime.date.today()`)와 형식이 맞지 않아 연산 에러가 발생했다. 이를 해결하기 위해 `datetime.datetime.strptime()` 메서드를 사용하여 입력받은 문자열을 정식 날짜 객체로 파싱하고, `.days` 속성을 통해 정확한 남은 일수를 구하는 방식을 학습하여 적용했다.
* **총평**: 화려한 UI를 가진 프로그램은 아니지만, 실생활에서 겪는 예산 관리 문제를 조건문과 라이브러리 활용이라는 기초 프로그래밍 논리를 통해 직접 해결해 보았다는 점에서 뜻깊은 프로젝트였다. 특히 고정된 상숫값을 비교하는 것이 아니라 사용자의 입력 날짜에 따라 실시간으로 변하는 예산 한도 알고리즘을 직접 설계하여 프로그램의 유연성을 높였다.
